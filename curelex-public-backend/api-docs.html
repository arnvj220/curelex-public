Curelex API Docs

Complete reference for the Curelex digital healthcare platform backend — authentication, appointments, prescriptions, and more.

Base URL http://localhost:5000/api
Format JSON
Auth Bearer JWT
Storage Cloudinary
🔐

Authentication

🔑 Bearer Token Required

Protected routes require a JWT token in the Authorization header. Obtain a token by logging in via POST /api/users/login.

Authorization: Bearer <your_jwt_token>
Admin routes additionally require the user's role to be "admin" in the database.
👤

Users

/api/users
POST /api/users/register Register a new user

Request Body

FieldTypeRules
nameString REQUIREDCannot be empty
emailString REQUIREDValid email format
passwordString REQUIREDMinimum 6 characters

Example Request

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "secret123"
}

Response — 201

{
  "message": "User registered successfully",
  "user": {
    "_id": "64a1f...",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "patient"
  }
}
POST /api/users/login Login and receive JWT token

Request Body

FieldTypeRules
emailString REQUIREDRegistered email
passwordString REQUIREDAccount password

Response — 200

{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI..."
}

Doctors

/api/doctors
POST /api/doctors/register Register a doctor (with certificate upload)
Content-Type: multipart/form-data — required for file upload

Request Fields

FieldTypeRules
nameString REQUIREDDoctor's full name
emailString REQUIREDValid unique email
specializationString REQUIREDe.g. Cardiology
experienceNumber REQUIREDYears of experience
certificateFile OPTIONALJPG, PNG or PDF — uploaded to Cloudinary

Response — 201

{
  "message": "Doctor registered successfully",
  "doctor": {
    "_id": "64b2e...",
    "name": "Dr. Sarah Khan",
    "verificationStatus": "pending",
    "certificateUrl": "https://res.cloudinary.com/..."
  }
}
GET /api/doctors/all Get all approved doctors

Response — 200

{
  "success": true,
  "count": 3,
  "doctors": [ { /* doctor objects */ } ]
}

Admin

/api/admin

⚠️ Admin Access Only

All admin routes require a valid JWT token AND the user's role must be "admin" in the database.

GET /api/admin/pending-doctors 🔐 AUTH ADMIN

Response — 200

[ { "verificationStatus": "pending", /* doctor fields */ } ]
POST /api/admin/approve/:id 🔐 AUTH ADMIN Approve a doctor

URL Params

ParamTypeDescription
:idString REQUIREDDoctor's MongoDB ObjectId

Response — 200

{ "message": "Doctor Approved", "doctor": { /* updated doctor */ } }
POST /api/admin/reject/:id 🔐 AUTH ADMIN Reject a doctor

URL Params

ParamTypeDescription
:idString REQUIREDDoctor's MongoDB ObjectId

Response — 200

{ "message": "Doctor Rejected", "doctor": { /* updated doctor */ } }

Appointments

/api/appointments
POST /api/appointments/book 🔐 AUTH Book an appointment

Request Body

FieldTypeRules
patientIdObjectId REQUIREDMust exist in Users
doctorIdObjectId REQUIREDDoctor must be approved
dateString REQUIREDe.g. "2026-03-20"
timeString REQUIREDe.g. "10:00 AM"
Smart Validations: Doctor must be verified · Patient must exist · Time slot must not be already booked

Example Request

{
  "patientId": "64a1f...",
  "doctorId": "64b2e...",
  "date": "2026-03-20",
  "time": "10:00 AM"
}

Response — 201

{
  "message": "Appointment booked successfully",
  "appointment": {
    "status": "pending",
    "date": "2026-03-20",
    "time": "10:00 AM"
  }
}
GET /api/appointments/patient/:id 🔐 AUTH Get all appointments for a patient

Response — 200

{
  "success": true,
  "count": 2,
  "appointments": [
    {
      "doctorId": { "name": "Dr. Khan", "specialization": "Cardiology" },
      "patientId": { "name": "John", "email": "john@..." }
    }
  ]
}
GET /api/appointments/doctor/:id 🔐 AUTH Get all appointments for a doctor

Response — 200

{
  "success": true,
  "count": 5,
  "appointments": [ { /* appointment objects with populated patient & doctor */ } ]
}

Request Body

FieldTypeRules

Prescriptions

/api/prescriptions
POST /api/prescriptions/add Create a new prescription

Request Body

FieldTypeRules
patientIdObjectId REQUIREDValid user ID
doctorIdObjectId REQUIREDValid doctor ID
medicinesArray REQUIREDMin 1 item
medicines[].nameString REQUIREDMedicine name
medicines[].dosageString REQUIREDe.g. "500mg"
medicines[].durationString REQUIREDe.g. "5 days"
notesString OPTIONALDoctor's notes

Example Request

{
  "patientId": "64a1f...",
  "doctorId": "64b2e...",
  "medicines": [
    {
      "name": "Paracetamol",
      "dosage": "500mg",
      "duration": "5 days"
    }
  ],
  "notes": "Take after meals"
}

Response — 201

{ "message": "Prescription added successfully", "prescription": { /* object */ } }
GET /api/prescriptions/patient/:id Get all prescriptions for a patient

Response — 200

{
  "success": true,
  "count": 1,
  "prescriptions": [ { /* populated doctor & patient info */ } ]
}
GET /api/prescriptions/doctor/:id Get all prescriptions by a doctor

Response — 200

{
  "success": true,
  "count": 3,
  "prescriptions": [ { /* populated doctor & patient info */ } ]
}

Status Codes & Errors

HTTP Status Codes

200OK — success
201Created — resource created
400Bad Request — validation failed
401Unauthorized — invalid/missing token
403Forbidden — not admin
404Not Found — resource missing
500Server Error — unexpected error

Error Response Format

{
  "success": false,
  "message": "Descriptive error message"
}

Validation Error Format

{
  "errors": [
    { "msg": "Name is required", "path": "name" },
    { "msg": "Valid email is required", "path": "email" }
  ]
}
